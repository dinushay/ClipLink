## 2024-05-18 - Actionable Empty States in Bot Commands
**Learning:** Empty states that only communicate the absence of data (e.g., "No streamers monitored") leave users at a dead end, particularly in a conversational UI like Discord where discoverability of commands isn't always obvious. Providing a clear "next step" or call-to-action directly in the empty state significantly reduces user friction.
**Action:** Always include a helpful call-to-action (like suggesting a specific command) when displaying an empty state or empty list in conversational bots.
